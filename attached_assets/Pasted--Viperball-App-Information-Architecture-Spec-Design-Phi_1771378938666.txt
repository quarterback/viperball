# Viperball App â€” Information Architecture Spec

## Design Philosophy

**SaaS patterns, not sports sim menus.**

Traditional sports sims (Football Manager, OOTP) bury information in nested menus. Users memorize where things live. We use modern SaaS patterns instead:

- Flat navigation (3-4 top-level sections)
- Command bar for instant access
- Contextual panels instead of page navigation
- Search as primary discovery
- Export-first data access

## Mobile Considerations

The IA works on mobile with these adjustments:

- **Bottom nav** instead of top tabs (League, My Team, Play, Export)
- **Full-screen panels** instead of slide-in
- **Command bar** via search icon, not keyboard shortcut
- **Swipe** to navigate between views in a section

---

## Implementation Notes

### For Streamlit (Quick Version)

- Use `st.tabs()` for top-level nav
- Use `st.expander()` or columns for panels
- Command bar = `st.text_input` with autocomplete logic
- Export = generate string, `st.download_button`

### For React (Full Version)

- React Router for top-level sections
- Zustand/Redux for dynasty state
- Slide-in panels via Framer Motion or Radix
- Command bar via cmdk library
- Export via file download or clipboard API
